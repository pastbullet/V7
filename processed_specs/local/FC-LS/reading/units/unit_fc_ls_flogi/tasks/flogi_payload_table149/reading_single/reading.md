Read Table 149 and lead-in text on pages 165-166. The lead-in says Table 149 defines the payload format for FLOGI and PLOGI ELSs and LS_ACCs, and that the parameter definitions apply to PLOGI, FLOGI, PLOGI LS_ACC, and FLOGI LS_ACC unless stated otherwise. It also states there are no separate Class 6 login Service Parameters because Class 1 service parameters are used instead. Table 149 lays out the payload by word range: Word 0 ELS_Command code; Words 1-4 Common Service Parameters (16 bytes); Words 5-6 Port_Name; Words 7-8 Node_ or Fabric_Name; Words 9-12 Class 1 and Class 6 Service Parameters (16 bytes); Words 13-16 Class 2 Service Parameters (16 bytes); Words 17-20 Class 3 Service Parameters (16 bytes); Words 21-24 Obsolete; Words 25-28 Vendor Version Level (16 bytes); Words 29-30 Services Availability (8 bytes); Word 31 Login Extension Data Length; Words 32-61 Reserved; Words 62-63 Clock Synchronization QoS (8 bytes); Words 64 to n Login Extension Data (if any). Footnote a grounds the conditional presence rule for Words 29-31 and 62-64 to n: these fields are only present when the Payload Bit is set to one; when the Payload Bit is zero, those fields are absent and the payload is 116 bytes long. The pre-table text also gives a procedural condition: if a Login of 256 bytes or more is required and buffer conditions are unknown, first issue Login with Payload Bit zero, then if the Query Buffer Conditions bit is one in LS_ACC issue RPBC, and if both receive-data-size fields are at least 256 bytes issue Login with Payload Bit one.
