Section 6.6.1 states that Table 149 defines the payload format for FLOGI and PLOGI ELSs and LS_ACCs, and that the parameter definitions apply to PLOGI, FLOGI, PLOGI LS_ACC, and FLOGI LS_ACC unless stated otherwise. Table 149 lays out the payload by word positions: word 0 is `ELS_Command code`; words 1-4 are `Common Service Parameters (16 bytes)`; words 5-6 are `Port_Name`; words 7-8 are `Node_ or Fabric_Name`; words 9-12 are `Class 1 and Class 6 Service Parameters (16 bytes)`; words 13-16 are `Class 2 Service Parameters (16 bytes)`; words 17-20 are `Class 3 Service Parameters (16 bytes)`; words 21-24 are `Obsolete`; words 25-28 are `Vendor Version Level (16 bytes)`; words 29-30 are `Services Availability (8 bytes)`; word 31 is `Login Extension Data Length`; words 32-61 are `Reserved`; words 62-63 are `Clock Synchronization QoS (8 bytes)`; and words 64 to n are `Login Extension Data (if any)`. The footnote on the table says `Services Availability`, `Login Extension Data Length`, `Clock Synchronization QoS`, and `Login Extension Data (if any)` are only present when the Payload Bit is set to one; if the Payload Bit is zero, those fields are absent and the payload is 116 bytes long. The preceding text gives a login procedure for large payloads: first send Login with Payload Bit zero; if the LS_ACC has Query Buffer Conditions bit one, issue RPBC; if both the RPBC LS_ACC ELS Receive Data Field Size and the Login LS_ACC Buffer-to-buffer Receive Data_Field Size are at least 256 bytes, issue Login with Payload Bit one. Section 6.6.1 also states that there are no separate Class 6 login Service Parameters because Class 1 service parameters are used instead.
